Binary Search Easy LC #704
Classic Template Foundation
🧭
Approach
Use three pointers: left, right, mid. Set left = 0, right = n-1. While left <= right, compute mid and compare nums[mid] to target. If equal → return. If target is larger → move left = mid + 1. If smaller → move right = mid - 1.
💡
Tip
Always compute mid as left + (right - left) / 2 — not (left + right) / 2. The second form overflows Java's int when indices exceed ~2.1B. The <= in the while loop (not <) ensures the single-element window is checked.
⚡
FAANG Tip
Classic opener. Every binary search variant descends from this template. Edge cases to nail: empty array → return -1, single element (target present/absent), target smaller than all elements, larger than all elements. Interviewers use this to calibrate whether you know mid+1 vs mid pointer movement cold.
📄 Solution
Time: O(log n) Space: O(1)
▾
Java 
public int search(int[] nums, int target) {
    int left = 0, right = nums.length - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;  // overflow-safe
        if (nums[mid] == target)      return mid;
        else if (nums[mid] < target)  left  = mid + 1;
        else                           right = mid - 1;
    }
    return -1;
}

First Bad Version Easy LC #278
Left Boundary Predicate Search
🧭
Approach
Find the first position where a boolean predicate becomes true. Use left < right (not <=). When isBadVersion(mid) is true, set right = mid — mid could be the answer, don't exclude it. When false, set left = mid + 1. Loop converges when left == right.
💡
Tip
When you find a satisfying mid, use right = mid (not mid - 1) to keep it as a candidate. The left < right condition guarantees termination — when they meet, that's your answer. Return left (equals right at this point).
⚡
FAANG Tip
Template for "find first X". This pattern covers: first bad version, first 1 in sorted 0/1 array, leftmost position >= target. Edge cases: n = 1 (always bad version 1), all versions bad (answer is 1), only last version bad (answer is n).
📄Solution
Time: O(log n) Space: O(1)
▾
Java
public int firstBadVersion(int n) {
    int left = 1, right = n;
    while (left < right) {          // Note: < not <=
        int mid = left + (right - left) / 2;
        if (isBadVersion(mid)) right = mid;    // could be answer
        else                   left  = mid + 1;
    }
    return left;                   // left == right == answer
}

Sqrt(x) Easy LC #69
Right BoundaryMath
🧭
Approach
Binary search over the answer space [1, x]. Track the last valid mid where mid*mid <= x as the candidate answer. If mid*mid == x return immediately. If mid*mid < x save mid as candidate and search right. If mid*mid > x search left.
💡
Tip
Use (long) mid * mid cast to avoid int overflow when x is close to Integer.MAX_VALUE. The answer is always the floor — last position where mid² ≤ x.
⚡
FAANG Tip
Answer-space search. You're not searching the input — you're searching the answer range. Edge cases: x = 0 → return 0, x = 1 → return 1. Watch for the long cast — missing it is a common bug.
📄Solution
Time: O(log x)Space: O(1)
▾
Java
public int mySqrt(int x) {
    if (x < 2) return x;
    int left = 1, right = x / 2, ans = 0;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        long sq = (long) mid * mid;   // avoid overflow
        if      (sq == x) return mid;
        else if (sq <  x) { ans = mid; left  = mid + 1; }
        else               right = mid - 1;
    }
    return ans;
}

Search in Rotated Sorted ArrayMediumLC #33
Rotated ArrayTwo Halves
🧭
Approach
One half is always sorted. Identify which half by comparing nums[left] with nums[mid]. If left half is sorted and target falls in it → search left. Otherwise search right. Apply same logic symmetrically for right half sorted.
💡
Tip
Check nums[left] <= nums[mid] to identify the sorted half. Then ask: does target fall in [nums[left], nums[mid])? If yes → go left. If no → go right. Draw this on paper — it becomes muscle memory fast.
⚡
FAANG Tip
Google/Meta favourite. The key insight: in a rotated array, at least one half is always sorted — use that as your anchor. Edge cases: array not rotated at all (normal BS), single element, target at rotation point, two-element arrays.
📄Solution
Time: O(log n)Space: O(1)
▾
Java
public int search(int[] nums, int target) {
    int left = 0, right = nums.length - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] == target) return mid;

        if (nums[left] <= nums[mid]) {            // left half sorted
            if (nums[left] <= target && target < nums[mid])
                right = mid - 1;
            else
                left  = mid + 1;
        } else {                                    // right half sorted
            if (nums[mid] < target && target <= nums[right])
                left  = mid + 1;
            else
                right = mid - 1;
        }
    }
    return -1;
}

Find Minimum in Rotated Sorted ArrayMediumLC #153
Rotated ArrayMinimum
🧭
Approach
Compare nums[mid] with nums[right] (not left). If nums[mid] > nums[right], the minimum is in the right half → move left = mid + 1. Otherwise minimum is in left half including mid → move right = mid. Loop until left == right.
💡
Tip
Always compare with nums[right] here — comparing with nums[left] is trickier. The minimum is always on the "unsorted" side. Use left < right loop condition — converges to the minimum index.
⚡
FAANG Tip
Pair with LC #33. Together they cover all rotated array variants. Distinguish this from LC #154 (duplicates allowed) — that requires handling nums[mid] == nums[right] by shrinking right. Edge case: array not rotated → first element is minimum.
📄Solution
Time: O(log n)Space: O(1)
▾
Java
public int findMin(int[] nums) {
    int left = 0, right = nums.length - 1;
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] > nums[right])
            left  = mid + 1;   // min is in right half
        else
            right = mid;        // mid could be min
    }
    return nums[left];
}

Search in Rotated Array II (Duplicates)MediumLC #81
Rotated ArrayDuplicates
🧭
Approach
Same as LC #33 but add one extra case: when nums[left] == nums[mid] == nums[right], you cannot determine which half is sorted. Safe move: shrink both ends (left++; right--). All other cases identical to LC #33.
💡
Tip
The ambiguity case nums[left] == nums[mid] forces linear fallback. That's why worst case degrades to O(n) — e.g. array [1,1,1,1,1,2,1]. Average is still O(log n).
⚡
FAANG Tip
Follow-up to LC #33. Interviewers often ask: "What if there are duplicates?" Know this answer cold. The key insight is that duplicates break the ability to determine the sorted half — so you safely shrink both ends and pay O(n) worst case.
📄Solution
Time: O(n) worstSpace: O(1)
▾
Java
public boolean search(int[] nums, int target) {
    int left = 0, right = nums.length - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] == target) return true;
        // ambiguous — shrink both ends safely
        if (nums[left] == nums[mid] && nums[mid] == nums[right]) {
            left++; right--;
        } else if (nums[left] <= nums[mid]) {
            if (nums[left] <= target && target < nums[mid])
                right = mid - 1;
            else left  = mid + 1;
        } else {
            if (nums[mid] < target && target <= nums[right])
                left  = mid + 1;
            else right = mid - 1;
        }
    }
    return false;
}

Koko Eating BananasMediumLC #875
Answer Space BSFeasibility
🧭
Approach
Binary search on the answer (speed k) from [1, max(piles)]. For each candidate mid, check if Koko can finish in h hours using a helper. If feasible, try slower (search left). If not feasible, must go faster (search right).
💡
Tip
Ceiling division trick: hours += (pile + k - 1) / k — avoids floating point. The answer space upper bound is max(piles) because at that speed every pile finishes in exactly 1 hour.
⚡
FAANG Tip
Template for "minimise the maximum". Once you see this pattern, it covers: Ship Packages (LC #1011), Split Array (LC #410), Minimum Days to Make Bouquets (LC #1482). The structure is always: define answer space → binary search → write a greedy feasibility checker.
📄Solution
Time: O(n log m)Space: O(1)
▾
Java
public int minEatingSpeed(int[] piles, int h) {
    int left = 1;
    int right = Arrays.stream(piles).max().getAsInt();
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (canFinish(piles, mid, h)) right = mid;
        else                           left  = mid + 1;
    }
    return left;
}
private boolean canFinish(int[] piles, int k, int h) {
    int hours = 0;
    for (int p : piles)
        hours += (p + k - 1) / k;  // ceiling division
    return hours <= h;
}

Capacity to Ship PackagesMediumLC #1011
Answer Space BSGreedy Check
🧭
Approach
Binary search on capacity [max(weights), sum(weights)]. Min = heaviest item (can't skip it). Max = ship everything at once. For each candidate capacity, greedily simulate shipping days. If days ≤ D → feasible, try lower capacity.
💡
Tip
Left bound = max(weights) not 1 — you must be able to carry the heaviest package. Right bound = sum(weights) — one trip carries everything. Same structural template as Koko, different feasibility check.
⚡
FAANG Tip
Identical structure to Koko. Once you've solved Koko, this is ~5 minutes. The pattern: set left/right bounds thoughtfully, binary search, write canShip greedy. Amazon asks this frequently — recognise the "minimise max capacity" framing immediately.
📄Solution
Time: O(n log W)Space: O(1)
▾
Java
public int shipWithinDays(int[] weights, int days) {
    int left = 0, right = 0;
    for (int w : weights) {
        left  = Math.max(left, w);   // must carry heaviest
        right += w;                   // worst case: one trip
    }
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (canShip(weights, days, mid)) right = mid;
        else                             left  = mid + 1;
    }
    return left;
}
private boolean canShip(int[] weights, int days, int cap) {
    int d = 1, cur = 0;
    for (int w : weights) {
        if (cur + w > cap) { d++; cur = 0; }
        cur += w;
    }
    return d <= days;
}

Find Peak ElementEasyLC #162
Peak / Local MaxSlope Logic
🧭
Approach
No sorted array needed. Compare nums[mid] vs nums[mid+1]. If ascending slope (nums[mid] < nums[mid+1]) → peak is on the right → left = mid + 1. If descending slope → peak is on the left including mid → right = mid.
💡
Tip
Mental model: nums[-1] = nums[n] = -∞. A peak must exist by this boundary assumption. If you're on an upslope, there's a peak ahead. Use left < right loop — terminates at a single peak index.
⚡
FAANG Tip
Classic search without sorted array. Key insight: if nums[mid] < nums[mid+1], a peak MUST exist to the right — the boundary guarantee nums[n] = -∞ means the rising function must eventually fall. Edge cases: single element (it's the peak), all ascending, all descending.
📄Solution
Time: O(log n)Space: O(1)
▾
Java
public int findPeakElement(int[] nums) {
    int left = 0, right = nums.length - 1;
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] > nums[mid + 1])
            right = mid;        // peak is left (incl. mid)
        else
            left  = mid + 1;   // peak is right
    }
    return left;
}

Search a 2D MatrixEasyLC #74
2D MatrixIndex Mapping
🧭
Approach
Treat the m×n matrix as a flat sorted array of length m*n. Binary search on [0, m*n-1]. Map mid back to 2D: row = mid / n, col = mid % n. Standard binary search from there.
💡
Tip
Index mapping: matrix[mid / cols][mid % cols] — this is the key insight. Works because each row starts where the previous row ends (guaranteed by problem constraints). Clean, elegant, no nested loops.
⚡
FAANG Tip
Shows index arithmetic mastery. Interviewers love this — it demonstrates you can think beyond 1D arrays. Note: LC #240 (Search a 2D Matrix II) is different — rows and columns sorted but rows not contiguous, requires a different O(m+n) approach starting from top-right.
📄Solution
Time: O(log mn)Space: O(1)
▾
Java
public boolean searchMatrix(int[][] matrix, int target) {
    int m = matrix.length, n = matrix[0].length;
    int left = 0, right = m * n - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        int val = matrix[mid / n][mid % n];  // key mapping
        if      (val == target) return true;
        else if (val <  target) left  = mid + 1;
        else                    right = mid - 1;
    }
    return false;
}

Find K Closest ElementsMediumLC #658
Window SearchK Closest
🧭
Approach
Binary search for the left start of the k-element window. Search space is [0, n-k]. Compare distances: x - arr[mid] vs arr[mid+k] - x. If left distance is greater → window should slide right (left = mid+1). Otherwise → right = mid.
💡
Tip
You're not searching for x — you're searching for the left boundary of the k-window. right = arr.length - k because window of size k starting at right would be the last valid window. No absolute values needed in comparison.
⚡
FAANG Tip
O(log(n-k)) vs naive O(n). Shows mastery of searching a non-obvious search space. Interviewers often ask "can you do better than O(n)?" — this is the answer. Edge cases: x smaller than all elements (return first k), x larger than all (return last k).
📄Solution
Time: O(log(n-k))Space: O(1)
▾
Java
public List<Integer> findClosestElements(int[] arr, int k, int x) {
    int left = 0, right = arr.length - k;   // window start range
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (x - arr[mid] > arr[mid + k] - x)
            left  = mid + 1;
        else
            right = mid;
    }
    List<Integer> result = new ArrayList<>();
    for (int i = left; i < left + k; i++) result.add(arr[i]);
    return result;
}

Median of Two Sorted ArraysHardLC #4
HardPartitionTwo Arrays
🧭
Approach
Binary search on a partition of the smaller array. For each partition i of A, compute partition j of B such that left halves balance right halves. Invariant: maxLeftA <= minRightB AND maxLeftB <= minRightA. When invariant holds, compute median from the 4 boundary values.
💡
Tip
Always binary search on the shorter array — swap if needed. Use Integer.MIN_VALUE/MAX_VALUE as sentinels for empty partition edges. j = (m + n + 1) / 2 - i ensures left halves always total (m+n+1)/2 elements.
⚡
FAANG Tip
Top FAANG hard — Google, FB, Amazon flagship. Practice explaining the partition invariant verbally — that's what separates candidates. Know: O(log(min(m,n))) is the target, naive merge is O(m+n). Draw the partition on paper. Edge cases: empty array, one element in each, all of A < all of B.
📄Solution
Time: O(log min(m,n))Space: O(1)
▾
Java
public double findMedianSortedArrays(int[] A, int[] B) {
    if (A.length > B.length) return findMedianSortedArrays(B, A);
    int m = A.length, n = B.length;
    int left = 0, right = m;
    while (left <= right) {
        int i = left + (right - left) / 2;   // partition A
        int j = (m + n + 1) / 2 - i;           // partition B

        int maxLA  = (i == 0) ? Integer.MIN_VALUE : A[i-1];
        int minRA  = (i == m) ? Integer.MAX_VALUE : A[i];
        int maxLB  = (j == 0) ? Integer.MIN_VALUE : B[j-1];
        int minRB  = (j == n) ? Integer.MAX_VALUE : B[j];

        if (maxLA <= minRB && maxLB <= minRA) {
            if ((m + n) % 2 == 1)
                return Math.max(maxLA, maxLB);
            return (Math.max(maxLA, maxLB) + Math.min(minRA, minRB)) / 2.0;
        } else if (maxLA > minRB) right = i - 1;
        else                        left  = i + 1;
    }
    throw new IllegalArgumentException();
}
